# intelligent-chatbot-analysis
顧客の会話からニーズを抽出するインテリジェント・チャットボットシステム
 課題テーマ：インテリジェント・チャットボット分析システム
【着眼点】
BtoB SaaS企業のCS（カスタマーサクセス）において、単なるFAQ対応ではなく、「顧客の不満や期待をデータ化すること」がビジネス成長の鍵であると考えました。人間では見落としがちな感情の機微をAIでスコアリングし、マーケティング戦略にフィードバックできる仕組みを目指しました。

【アプローチ】
リアルタイム分析: ユーザーの入力と同時に感情分析とニーズ抽出を行い、即座にインサイトを構造化データとして生成。
対話と分析の統合: 1つのプロンプトで「CSとしての回答」と「マーケティング用分析」を同時に行うことで、処理効率を最適化。

【工夫した点・苦労した点】
技術的適応力: 開発中に直面した「APIモデルのバージョン差異による404エラー」に対し、環境に依存せず利用可能なモデルを動的に判別・選択する「自動モデルセレクト・ロジック」を独自に実装し、システムの堅牢性を高めました。
プロンプトエンジニアリング: AIにCS担当としてのペルソナ（人格）を厳格に持たせ、一貫性のあるブランド体験を提供できるよう調整しました。

【使用した技術】
Python / FastAPI: 軽量かつ高速なAPIサーバー構築。
Google Gemini 1.5 Flash: 最新の大規模言語モデルによる高度な文脈理解。
python-dotenv: セキュリティを考慮したAPIキーの秘匿管理。
